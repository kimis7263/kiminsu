🚀 프로젝트 보고서: AI Log Visualizer
(Master & Commander Edition)

1. 기획 (어떤 문제를 왜 해결하려 했는가)

AI 모델의 하이퍼파라미터를 튜닝하며 학습을 진행할 때마다 수많은 CSV 형태의 로그(Loss, Accuracy 등) 파일이 생성됩니다. 기존에는 여러 모델의 성능을 비교하기 위해 매번 엑셀을 열어 차트를 삽입하거나, Jupyter Notebook을 켜서 파이썬 코드를 작성해야 했습니다. 이는 연구자들의 흐름을 심각하게 끊는 비효율적인 반복 작업이었습니다.

이를 해결하기 위해 코딩 없이 탐색기에서 파일을 드래그 앤 드롭하는 것만으로 여러 모델의 로그를 즉각적으로 겹쳐보고(Overlay), 인터랙티브하게 분석할 수 있는 현업 연구자용 윈도우 네이티브 유틸리티를 개발하고자 했습니다.

2. 구성물 (주요 기능 목록)

  • Multi-File Overlay (누적 비교): 여러 개의 CSV 파일 데이터를 하나의 그래프 도화지에 누적하여 그려내는 핵심 기능
  • 인터랙티브 플로팅 범례 (Legend): 마우스 우클릭으로 드래그하여 화면 내에서 자유롭게 이동하고, 클릭하여 그래프의 이름을 즉시 수정할 수 있는 커스텀 범례 패널
  • 스무딩(Moving Average) 및 통계 패널: 노이즈가 심한 로그를 슬라이더로 부드럽게 펴고, 해당 데이터의 Min/Max/Mean/Final 값을 실시간 테이블로 제공
  • 그래프 2D 메모(주석): 논문이나 보고서 작성을 위해 그래프 특정 좌표에 커스텀 텍스트 메모를 직접 삽입하고 색상을 지정하는 기능
  • 미니 플로팅 모드: 프로그램 창을 바탕화면에 떠다니는 작은 아이콘으로 전환하여 작업 공간을 절약하는 기능
  • 프로페셔널 Export: 고화질 PNG 캡처, 논문용 벡터 이미지(SVG) 저장 기능 지원
  • 다크/라이트 테마 전환: 윈도우 네이티브 UI의 깨짐 없이 그래프와 전체 창의 테마를 부드럽게 전환

3. 실현 방법 (사용 기술, AI 도구 활용 과정)

본 프로그램은 Python과 네이티브 윈도우 UI 구성을 위한 PySide6 (Qt) 프레임워크를 기반으로 개발되었습니다. 특히 수십만 개의 데이터 포인트를 렉 없이 렌더링하고, 다크모드 배경 전환을 완벽히 지원하기 위해 하드웨어 가속 기반의 PyQtGraph를 활용했습니다. 데이터 처리는 Pandas를 사용했습니다.

💡 AI 도구 활용 과정 (바이브 코딩): 기획의 뼈대를 잡은 후 LLM과의 지속적인 상호작용을 통해 코드를 발전시켰습니다. 특히 'PyInstaller 패키징 시 환경변수 오류 해결', '우클릭으로 드래그되는 플로팅 위젯의 좌표 계산 로직 구현', '테마 전환 시 PyQtGraph 초기 배경색 버그(검은 화면) 해결' 등 GUI 프로그래밍에서 까다로운 트러블슈팅과 세밀한 UI/UX 조정을 AI와 함께 신속하게 해결하며 완성도를 극대화했습니다.

5. 특장점 (이 프로그램만의 강점)

가장 큰 강점은 현업 데이터 사이언티스트들의 워크플로우를 완벽히 이해한 '극강의 편의성(UX)'입니다. 귀찮은 폴더 찾기 없이 드래그 앤 드롭으로 파일을 열고, 그래프가 가려지지 않게 범례를 우클릭으로 치우고, 프로그램 화면 내에서 직접 텍스트를 입력해 캡처(SVG/PNG)까지 한 번에 끝낼 수 있습니다. 엑셀이나 별도의 시각화 툴을 켤 필요 없이 이 가벼운 유틸리티 하나로 AI 로그 분석과 보고서용 이미지 추출을 모두 마칠 수 있습니다.

6. 한계점 (현재 버전의 한계)

현재 버전은 .csv 확장자의 정적(Static) 파일 분석에 특화되어 있습니다. 따라서 딥러닝 프레임워크(PyTorch, TensorFlow)에서 학습과 동시에 생성하는 TensorBoard 이벤트 파일(.tfevents)을 직접 읽어들이는 기능은 탑재되어 있지 않습니다. 또한 실시간으로 갱신되는 파일을 모니터링하는 데는 수동 새로고침(재로드)이 필요합니다.

7. Todo (향후 개선 방향)

  • TensorBoard Native 포맷 지원: 추가적인 변환 작업 없이 .tfevents 로그 파일을 직접 파싱할 수 있는 엔진 탑재
  • 데이터베이스 연동: 로컬 파일뿐만 아니라 Weights & Biases(Wandb) 등 클라우드 로깅 플랫폼의 API를 직접 연결하여 데이터를 끌어오는 기능 추가
